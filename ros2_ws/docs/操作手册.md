## 一、项目概述

### 1.1 项目简介

本项目是第二十届全国大学生智能汽车竞赛-地瓜机器人智慧医疗挑战赛的参赛方案，基于ROS2架构实现智能医疗服务机器人功能。

### 1.2 核心功能

| 功能模块 | 说明 | 对应文件 |
|---------|------|---------|
| 二维码识别 | 识别任务发布点二维码 | qr_detector_node.py |
| 人形检测 | YOLOv8人形目标检测 | human_detector_node.py |
| 大模型图生文 | 图像描述生成 | llm_interface_node.py |
| 路径规划 | 顺时针/逆时针路径 | path_planner_node.py |
| 运动控制 | 速度与转向控制 | motion_controller_node.py |
| 激光避障 | 障碍物检测与躲避 | obstacle_avoidance_node.py |
| 语音播报 | TTS语音输出 | voice_output_node.py |
| 任务管理 | 整体流程调度 | task_manager_node.py |

### 1.3 硬件要求

- **主控**: RDK X5 (推荐)
- **底盘**: OriginCar / OriginCar Pro
- **传感器**: 单目摄像头、深度摄像头、单线激光雷达、IMU
- **电源**: 12V锂电池组

## 三、系统部署与运行

### 3.1 环境准备

#### 硬件连接
```
1. 将RDK X5与OriginCar底盘连接
2. 连接摄像头、激光雷达、IMU等传感器
3. 确保电源连接稳定
4. 启动机器人电源
```

#### 软件环境
```bash
# 1. 安装Ubuntu 22.04 (推荐)

# 2. 安装ROS2 Humble
sudo apt update && sudo apt install ros-humble-desktop

# 3. 创建工作空间
mkdir -p ~/ros2_ws/src
cd ~/ros2_ws/src

# 4. 克隆项目代码
git clone <您的仓库地址>

# 5. 安装Python依赖
cd ~/ros2_ws
pip install -r requirements.txt
```

### 3.2 编译项目

```bash
cd ~/ros2_ws

# 设置环境变量
source /opt/ros/humble/setup.bash

# 编译所有功能包
colcon build --symlink-install

# 设置工作空间环境变量
source install/setup.bash
```

### 3.3 启动系统

#### 方式一：完整启动（推荐）
```bash
ros2 launch task_manager competition.launch.py
```

#### 方式二：分模块启动
```bash
# 终端1：启动二维码识别
ros2 launch qr_detector qr_detector.launch.py

# 终端2：启动人形检测
ros2 launch human_detector human_detector.launch.py

# 终端3：启动导航
ros2 launch navigation navigation.launch.py

# 终端4：启动语音播报
ros2 launch voice_output voice_output.launch.py

# 终端5：启动任务管理（最后启动）
ros2 launch task_manager task_manager.launch.py
```

### 3.4 关闭系统

```bash
# 方式一：Ctrl+C 终止当前运行的launch
Ctrl+C

# 方式二：停止所有ROS2节点
ros2 node list | xargs ros2 node kill

# 关闭机器人电源
# 断开电源开关
```

## 四、功能模块操作说明

### 4.1 二维码识别模块

#### 功能说明
- 实时检测摄像头图像中的二维码
- 解析任务信息（顺时针/逆时针）
- 发布任务话题供其他模块使用

#### 话题接口
| 话题名称 | 类型 | 说明 |
|---------|------|------|
| `/camera/image_raw` | Image | 摄像头输入 |
| `/qr_task` | String | 二维码任务信息 |

#### 操作示例
```bash
# 查看二维码识别结果
ros2 topic echo /qr_task

# 发布测试任务（调试用）
ros2 topic pub /qr_task std_msgs/msg/String "data: '顺时针'"
```

### 4.2 人形检测模块

#### 功能说明
- 使用YOLOv8模型检测人形目标
- 裁剪并编码检测到的人形图像
- 发送至大模型进行描述

#### 话题接口
| 话题名称 | 类型 | 说明 |
|---------|------|------|
| `/camera/image_raw` | Image | 摄像头输入 |
| `/human_detected` | Image | 检测到的人形图像 |
| `/human_description` | String | 大模型描述结果 |

#### 操作示例
```bash
# 查看人形检测结果
ros2 topic echo /human_description

# 查看检测图像（需要rviz）
ros2 run rviz2 rviz2
```

### 4.3 导航避障模块

#### 功能说明
- 路径规划：根据任务生成顺时针/逆时针路径
- 运动控制：P控制实现路径跟踪
- 激光避障：实时检测并躲避障碍物

#### 话题接口
| 话题名称 | 类型 | 说明 |
|---------|------|------|
| `/scan` | LaserScan | 激光雷达数据 |
| `/cmd_vel` | Twist | 速度指令输出 |
| `/waypoints` | PoseArray | 路径点序列 |

#### 参数配置
编辑 `navigation_params.yaml`:
```yaml
safety_distance: 0.25      # 安全距离（车身宽度的1.5倍）
max_speed: 0.3            # 最大线速度
target_position_tolerance: 0.3  # 目标点容差
```

#### 操作示例
```bash
# 查看当前路径点
ros2 topic echo /waypoints

# 查看激光雷达数据
ros2 topic echo /scan --noarr

# 查看速度指令
ros2 topic echo /cmd_vel
```

### 4.4 语音播报模块

#### 功能说明
- 接收文本消息并转换为语音
- 支持Linux(espeak)和Windows(System.Speech)

#### 话题接口
| 话题名称 | 类型 | 说明 |
|---------|------|------|
| `/voice_message` | String | 待播报的文本消息 |

#### 操作示例
```bash
# 发送测试语音消息
ros2 topic pub /voice_message std_msgs/msg/String "data: '您好，欢迎使用智慧医疗机器人'"
```

### 4.5 任务管理模块

#### 功能说明
- 协调各模块完成比赛任务
- 状态管理：待机→前往任务点→执行任务→返回起点→完成
- 时间管理：确保在180秒内完成任务

#### 状态机
```
待机状态
    ↓ (收到二维码任务)
前往任务点状态
    ↓ (到达任务点)
执行任务状态
    ↓ (任务完成)
返回起点状态
    ↓ (到达起点)
完成状态
```

#### 话题接口
| 话题名称 | 类型 | 说明 |
|---------|------|------|
| `/task_status` | String | 当前任务状态 |
| `/task_progress` | Float32 | 任务进度(0-100) |

#### 操作示例
```bash
# 查看任务状态
ros2 topic echo /task_status

# 查看任务进度
ros2 topic echo /task_progress

# 重置任务（调试用）
ros2 service call /reset_task std_srvs/srv/Empty
```

## 五、调试与故障排除

### 5.1 常见问题排查

| 问题现象 | 可能原因 | 解决方法 |
|---------|---------|---------|
| 节点无法启动 | 依赖未安装 | 运行 `pip install -r requirements.txt` |
| 摄像头无图像 | 设备未连接 | 检查USB连接，确认设备权限 |
| 激光雷达无数据 | 驱动未加载 | 检查串口连接，确认波特率 |
| 机器人不动 | 电机未使能 | 检查电源和电机驱动 |
| 语音无输出 | TTS引擎问题 | 安装espeak或检查系统语音设置 |
| 二维码识别失败 | 距离过远或角度过大 | 调整摄像头位置和角度 |
| 人形检测失败 | 模型未加载 | 检查模型路径和网络连接 |
| 导航偏差大 | 参数未调优 | 调整kp_linear和kp_angular参数 |

### 5.2 日志查看

```bash
# 查看所有节点日志
ros2 topic echo /rosout

# 查看特定节点日志
ros2 logs get qr_detector_node

# 启动节点时输出详细日志
ros2 run qr_detector qr_detector_node --ros-args --log-level debug
```

### 5.3 参数调整

#### 导航参数调整
```bash
# 动态调整参数（无需重启）
ros2 param set /motion_controller_node kp_linear 0.8
ros2 param set /motion_controller_node max_speed 0.4

# 保存参数到文件
ros2 param dump /motion_controller_node -o params.yaml
```

#### 常用参数说明
| 参数 | 说明 | 推荐范围 |
|------|------|---------|
| kp_linear | 线速度P系数 | 0.3-1.0 |
| kp_angular | 角速度P系数 | 0.5-2.0 |
| max_speed | 最大线速度 | 0.2-0.5 m/s |
| safety_distance | 安全距离 | 0.2-0.5 m |

## 六、比赛现场操作流程

### 6.1 赛前准备

```
1. 硬件检查
   ├─ 电源电量检查（至少80%）
   ├─ 传感器连接检查
   └─ 网络连接检查

2. 软件准备
   ├─ 启动机器人
   ├─ 连接SSH（推荐使用VSCode远程连接）
   └─ 启动终端

3. 环境设置
   ├─ source /opt/ros/humble/setup.bash
   └─ cd ~/ros2_ws && source install/setup.bash

4. 测试运行
   ├─ 启动摄像头查看画面
   ├─ 测试激光雷达数据
   └─ 测试语音播报功能
```

### 6.2 比赛开始

```bash
# 方式一：使用launch文件（推荐）
ros2 launch task_manager competition.launch.py

# 方式二：使用脚本启动
./scripts/start_competition.sh
```

### 6.3 比赛中监控

```bash
# 打开多个终端监控
# 终端1：任务状态
ros2 topic echo /task_status

# 终端2：速度指令
ros2 topic echo /cmd_vel

# 终端3：二维码任务
ros2 topic echo /qr_task

# 终端4：人形检测结果
ros2 topic echo /human_description

# 终端5：激光雷达
ros2 topic echo /scan --noarr
```

### 6.4 紧急情况处理

```bash
# 紧急停止（机器人失控时）
ros2 topic pub /cmd_vel geometry_msgs/msg/Twist "linear:
  x: 0.0
  y: 0.0
  z: 0.0
angular:
  x: 0.0
  y: 0.0
  z: 0.0" -r 10

# 或直接断开电源
```

### 6.5 赛后数据记录

```bash
# 保存日志
ros2 bag record /qr_task /human_description /task_status /cmd_vel -O competition_recording

# 导出数据
ros2 bag info competition_recording
```
